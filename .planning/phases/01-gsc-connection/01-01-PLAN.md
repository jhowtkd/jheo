<action>
Create `gscRoutes(app: FastifyInstance)` following `channels.ts` + `settings.ts` patterns.

**GET /api/projects/:projectId/gsc/connection** (GSC-04, GSC-06)
- 404 if no row; verify project exists first (404 `project not found`)
- Never include `serviceAccountCiphertext` in response
- Response: `{ projectId, siteUrl, lastSyncAt, syncStatus, syncError, clientEmail: string | null }`
- On decrypt failure: update row `syncStatus = 'decrypt_error'`, `syncError = 'Encryption key changed — re-upload Service Account JSON'`, return 200 with that status (do not throw)
- If decrypt succeeds, extract `client_email` for response only

**PUT /api/projects/:projectId/gsc/connection** (GSC-01, GSC-02, GSC-03)
- Parse body with `PutGscConnectionBodySchema`; 400 on Zod failure
- Require `JHEO_SECRET_KEY`; 503 `{ error: 'JHEO_SECRET_KEY not set' }` if missing — mirror channels
- Validate SA JSON shape → call `testGscConnection(siteUrl, sa)` **before** persisting
- On test failure, return structured errors with `requestId: req.id` (per PATTERNS.md):
  - `permission_denied` → HTTP 403: `{ error: { code: 'gsc_permission_denied', message: 'Add {client_email} as user in GSC Settings → Users and permissions', requestId: req.id } }`
  - `site_not_found` → HTTP 404: `{ error: { code: 'gsc_site_not_found', message: 'Check siteUrl format (trailing slash for URL-prefix or sc-domain: prefix)', requestId: req.id } }`
  - `api_error` → HTTP 502: `{ error: { code: 'gsc_api_error', message, requestId: req.id } }`
- Do not save on test failure
- On test success: `encrypt(JSON.stringify(sa), secret)` → upsert `GscConnection` with `syncStatus: 'ok'`, `syncError: null`
- Return same safe status shape as GET (no ciphertext)

**DELETE /api/projects/:projectId/gsc/connection** (GSC-05)
- Delete `GscConnection` row only (no snapshot table yet)
- 404 if not connected; 200 on success

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API (PUT body) | Untrusted SA JSON and siteUrl from user upload |
| API → Google GSC | Outbound sites.get with SA credentials |
| API → DB | Encrypted ciphertext at rest; plaintext only in memory during request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | PUT /gsc/connection | mitigate | Validate SA JSON shape with Zod before use; never log private_key |
| T-01-02 | T | gsc-auth.ts sites.get | mitigate | Use encodeURIComponent for siteUrl path segment |
| T-01-03 | I | GET /gsc/connection | mitigate | Never return serviceAccountCiphertext; expose client_email only |
| T-01-04 | D | crypto.ts decrypt | mitigate | On failure set decrypt_error status; return 200 with hint, do not crash |
| T-01-05 | E | JHEO_SECRET_KEY | mitigate | 503 on write when secret unset; mirror channels pattern |
| T-01-06 | I | Error responses | mitigate | Include requestId on structured errors for audit trail |
</threat_model>
